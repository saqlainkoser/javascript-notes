## PART 1: Event Handling (Mouse, Keyboard, Form Events, Event Object, preventDefault)

### 1.1 What is an Event?

An event is any action the browser can detect — a click, a key press, a page load, a form submit. JavaScript "listens" for these and reacts.

**Real-world analogy:** A doorbell (event) triggers someone to open the door (handler function). You don't stand at the door all day checking — you _react_ when the bell rings.

```js
button.addEventListener('click', function () {
  console.log('Button was clicked!');
});
```

Three ways to attach handlers (mention pros/cons):

```js
// 1. Inline HTML (avoid — mixes JS with HTML)
<button onclick="sayHi()">Click</button>

// 2. Property assignment (only one handler allowed)
btn.onclick = sayHi;

// 3. addEventListener (best practice — multiple handlers allowed)
btn.addEventListener('click', sayHi);
```

---

### 1.2 Mouse Events

|Event|Fires when...|Real-world use|
|---|---|---|
|`click`|Mouse clicked|Buy button, menu toggle|
|`dblclick`|Double clicked|Zoom on image (Google Photos)|
|`mouseover` / `mouseout`|Pointer enters/leaves|Tooltips, dropdown menus|
|`mousedown` / `mouseup`|Button pressed/released|Drag-and-drop, drawing apps|
|`mousemove`|Pointer moves|Custom cursors, games|
|`contextmenu`|Right-click|Custom right-click menus (Figma, Gmail)|

**Real-world example — E-commerce "Quick View" on hover (like Amazon/Flipkart):**

```js
const productCard = document.querySelector('.product-card');
const quickView = document.querySelector('.quick-view');

productCard.addEventListener('mouseover', () => {
  quickView.style.display = 'block';
});
productCard.addEventListener('mouseout', () => {
  quickView.style.display = 'none';
});
```

---

### 1.3 Keyboard Events

|Event|Fires when...|Real-world use|
|---|---|---|
|`keydown`|Key pressed down|Shortcuts (Ctrl+S), game controls|
|`keyup`|Key released|Search-as-you-type, validation after typing|
|`keypress` (deprecated)|Character key pressed|— use `keydown` instead|

**Real-world example — Search bar like YouTube/Google (live suggestions):**

```js
const searchBox = document.querySelector('#search');

searchBox.addEventListener('keyup', (e) => {
  console.log('You typed:', e.target.value);
  console.log('Key pressed:', e.key);
});
```

**Real-world example — Keyboard shortcut (like Ctrl+Enter to send message in WhatsApp Web):**

```js
document.addEventListener('keydown', (e) => {
  if (e.ctrlKey && e.key === 'Enter') {
    sendMessage();
  }
});
```

---

### 1.4 Form Events

|Event|Fires when...|Real-world use|
|---|---|---|
|`submit`|Form is submitted|Login/signup forms|
|`change`|Input loses focus after value change|Dropdown selections|
|`input`|Value changes (every keystroke)|Live character counters|
|`focus` / `blur`|Element gains/loses focus|Highlighting active field|

```js
form.addEventListener('submit', function (e) {
  console.log('Form submitted!');
});
```

---

### 1.5 The Event Object

Every handler automatically receives an **event object** with useful info.

```js
button.addEventListener('click', function (event) {
  console.log(event.type);        // "click"
  console.log(event.target);      // exact element clicked
  console.log(event.clientX, event.clientY); // mouse position
  console.log(event.timeStamp);   // when it happened
});
```

**Real-world example — Event Delegation** (like a table with 100 delete buttons — don't attach 100 listeners, attach ONE on the parent):

```js
document.querySelector('#taskList').addEventListener('click', function (e) {
  if (e.target.classList.contains('delete-btn')) {
    e.target.closest('li').remove();
  }
});
```

This is how apps like Trello/Todoist handle dynamic lists efficiently.

---

### 1.6 preventDefault()

Stops the browser's **default behavior** for that event.

|Element|Default behavior|Why you'd prevent it|
|---|---|---|
|`<a href="">`|Navigates to link|Single Page Apps (React Router style navigation)|
|`<form>`|Reloads page on submit|Submit via JS/AJAX without page refresh|
|Right-click|Shows browser context menu|Custom menu (e.g., image protection sites)|

**Real-world example — Login form that submits without reloading (like Gmail):**

```js
loginForm.addEventListener('submit', function (e) {
  e.preventDefault(); // stop page reload
  const email = document.querySelector('#email').value;
  console.log('Logging in with:', email);
  // send data via fetch() instead of reloading
});
```

## PART 2: Forms, Form Validation, User Input Handling, Error Messages

### 2.1 Reading User Input

```js
const name = document.querySelector('#name').value;
const isChecked = document.querySelector('#agree').checked;
const selected = document.querySelector('#country').value;
```

### 2.2 Why Validate?

**Real-world analogy:** A bank won't let you submit a cheque with no amount written. Validation is the "sanity check" before data is accepted — same as signup forms on Instagram checking your email format before creating an account.

Two levels:

- **Client-side** (JS) — fast feedback, better UX (not secure alone)
- **Server-side** — the real security gate (always required too)

### 2.3 Common Validation Patterns

```js
function validateForm() {
  const name = document.querySelector('#name').value.trim();
  const email = document.querySelector('#email').value.trim();
  const password = document.querySelector('#password').value;
  let isValid = true;

  // Required field
  if (name === '') {
    showError('name', 'Name is required');
    isValid = false;
  }

  // Email pattern
  const emailPattern = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!emailPattern.test(email)) {
    showError('email', 'Enter a valid email address');
    isValid = false;
  }

  // Password length (like GitHub signup: min 8 chars)
  if (password.length < 8) {
    showError('password', 'Password must be at least 8 characters');
    isValid = false;
  }

  return isValid;
}
```

### 2.4 Showing Error Messages (UX pattern used by real sites)

```html
<input type="email" id="email">
<span class="error-msg" id="email-error"></span>
```

```js
function showError(fieldId, message) {
  const errorSpan = document.querySelector(`#${fieldId}-error`);
  errorSpan.textContent = message;
  document.querySelector(`#${fieldId}`).classList.add('input-error');
}

function clearError(fieldId) {
  document.querySelector(`#${fieldId}-error`).textContent = '';
  document.querySelector(`#${fieldId}`).classList.remove('input-error');
}
```

**Real-world example — Live validation as user types (like signup forms on most SaaS products):**

```js
document.querySelector('#email').addEventListener('input', function () {
  clearError('email');
  const emailPattern = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!emailPattern.test(this.value)) {
    showError('email', 'Invalid email format');
  }
});
```

### 2.5 HTML5 Built-in Validation (worth mentioning)

```html
<input type="email" required>
<input type="number" min="18" max="60" required>
<input type="text" pattern="[A-Za-z]{3,}" required>
```

Combine with JS for custom messages:

```js
input.addEventListener('invalid', function (e) {
  e.preventDefault();
  showError('email', 'Please enter a valid email');
});
```

## TASKS — Easy to Hard

### 🟢 Easy

1. **Color Changer:** Create a button that changes the background color of the page each time it's clicked (`click` event).
2. **Hover Card:** Make a `<div>` that shows "You're hovering!" text on `mouseover` and hides it on `mouseout`.
3. **Key Logger:** Create an input box that displays "You pressed: [key]" below it as the user types (`keyup`).
4. **Character Counter:** A textarea (max 100 chars) that shows "X characters remaining" live using the `input` event.
5. **Prevent Link Navigation:** A link that, when clicked, shows an alert instead of navigating (`preventDefault()`).

### 🟡 Medium

6. **Login Form (No Reload):** Build a form with email + password. On submit, `preventDefault()`, check both fields aren't empty, and log values to console instead of reloading the page.
7. **Simple Email Validator:** Validate an email field using regex; show a red border + error text if invalid, remove it once corrected.
8. **Show/Hide Password:** A checkbox that toggles an input's `type` between `password` and `text` (`change` event).
9. **Right-Click Menu Blocker:** Prevent the default browser right-click menu on an image and show a custom "Downloads disabled" message instead.
10. **Keyboard Shortcut:** Detect `Ctrl + S` anywhere on the page, `preventDefault()` the browser's save dialog, and show "Draft saved!" instead.

### 🔴 Hard

11. **Full Signup Form Validator:** Build a signup form (name, email, password, confirm password, age). Validate:
    
    - All fields required
    - Valid email format
    - Password ≥ 8 chars, at least 1 number
    - Confirm password matches
    - Age ≥ 18 Show inline error messages under each field, clear errors as the user corrects them (using `input` event), and only allow submission when all checks pass.
12. **Dynamic To-Do List with Event Delegation:** Build a to-do app where users add tasks (via form submit), and each task has a delete button. Use **event delegation** (one listener on the parent `<ul>`) to handle all delete clicks — don't attach listeners individually.
    
13. **Live Search Filter:** An input box that filters a list of 20+ items in real time as the user types (`input` event), showing "No results found" as a dynamic error message when nothing matches.
    
14. **Drag-to-Reorder List (Mouse Events):** Using `mousedown`, `mousemove`, and `mouseup`, let users drag list items to reorder them (no libraries).
    
15. **Multi-Step Form with Validation Per Step:** Build a 3-step form (Personal Info → Address → Payment). Each step validates its own fields before allowing "Next." Show step-specific error messages, and prevent form submission (`preventDefault()`) until the final step passes validation.
