
# JavaScript Functions — Complete Guide

## 1. What is a Function?

A function is a reusable block of code that performs a specific task. Instead of writing the same code multiple times, we write it once and call it whenever needed.

```javascript
function greet() {
  console.log("Hello");
}

greet();
greet();
greet();
```

**Output**

```
Hello
Hello
Hello
```


## 2. Why Do We Need Functions?

### Without Function

```javascript
console.log("Hello Ali");
console.log("Hello Sara");
console.log("Hello Ahmed");
```

### With Function

```javascript
function greet(name) {
  console.log("Hello " + name);
}

greet("Ali");
greet("Sara");
greet("Ahmed");
```

### Advantages

- Reusable
- Easy to maintain
- Less code
- Better readability
- Modular programming

## 3. Anatomy of a Function

```javascript
function add(a, b) {
  return a + b;
}
```

Let's understand every part:

|Part|Meaning|
|---|---|
|`function`|keyword|
|`add`|function name|
|`(a, b)`|parameters|
|`{ }`|function body|
|`return`|sends value back|
|`a + b`|returned value|

### Calling Function

```javascript
let result = add(10, 20);
console.log(result);
```

**Output**

```
30
```

## 6. Anonymous Function

Function without a name.

```javascript
const greet = function () {
  console.log("Hello");
};
```

Here `function (){}` is anonymous.

---

## 7. Named Function Expression

```javascript
const greet = function sayHello() {
  console.log("Hello");
};
```

- Outer `greet()` works.
- Inside, `sayHello()` also works.
- Outside, `sayHello()` doesn't work.

## 8. Arrow Function

Introduced in ES6.

### Normal Function

```javascript
function add(a, b) {
  return a + b;
}
```

### Arrow

```javascript
const add = (a, b) => {
  return a + b;
};
```

### Short Version

```javascript
const add = (a, b) => a + b;
```

## 13. Higher Order Function

A function that accepts a function and/or returns a function.

```javascript

  return operation(a, b);
}

function add(a, b) {
  return a + b;
}

console.log(calculator(add, 10, 20));
```

**Output**

```
30
```

## 17. Spread vs Rest

**Rest** — Collects values.

```javascript

```

**Spread** — Expands values.

```javascript

```

## 21. Closure ⭐⭐⭐⭐⭐⭐

> One of the most important concepts.
> 
> A closure is created when an inner function remembers variables from its outer function, even after the outer function has finished executing.

```javascript
function outer() {
  let count = 0;
  return function () {
    count++;
    console.log(count);
  };
}

const counter = outer();
counter();
counter();
counter();
```

**Output**

```
1
2
3
```

**Why?** The outer function finished, but `count` still exists because the inner function remembers it.

---

## 22. Lexical Scope

```javascript
let x = 100;

function outer() {
  let y = 200;
  function inner() {
    console.log(x);
    console.log(y);
  }
  inner();
}

outer();
```

**Output**

```
100
200
```

The inner function accesses parent variables.

## 23. Pure Function

Always returns the same output for the same input.

```javascript
function square(x) {
  return x * x;
}
```

Good.

---

## 24. Impure Function

Depends on outside data.

```javascript
let tax = 10;

function total(price) {
  return price + tax;
}
```

If `tax` changes, the result changes.

## 27. Constructor Function

Before ES6 classes, constructor functions were used to create objects.

```javascript
function Person(name, age) {
  this.name = name;
  this.age = age;
}

const p1 = new Person("Ali", 25);
console.log(p1);
```

**Output**

```
Person { name: "Ali", age: 25 }
```

## 28. Generator Function

Generator functions can pause and resume execution.

```javascript
function* numbers() {
  yield 1;
  yield 2;
  yield 3;
}

const gen = numbers();
console.log(gen.next());
console.log(gen.next());
console.log(gen.next());
console.log(gen.next());
```

**Output**

```
{ value: 1, done: false }
{ value: 2, done: false }
{ value: 3, done: false }
{ value: undefined, done: true }
```

## 30. The `this` Keyword in Functions

### Regular function

```javascript
const person = {
  name: "Ali",
  greet: function () {
    console.log(this.name);
  }
};

person.greet();
```

**Output:** `Ali`

### Arrow function

```javascript
const person = {
  name: "Ali",
  greet: () => {
    console.log(this.name);
  }
};

person.greet();
```

**Output (in most environments):** `undefined`

Arrow functions do not have their own `this`; they inherit it from the surrounding scope.

## 31. `call()`, `apply()`, and `bind()`

```javascript
function greet(city) {
  console.log(`Hello, I'm ${this.name} from ${city}`);
}

const person = { name: "Ali" };

greet.call(person, "Lahore");    // Hello, I'm Ali from Lahore
greet.apply(person, ["Karachi"]); // Hello, I'm Ali from Karachi

const bound = greet.bind(person, "Delhi");
bound(); // Hello, I'm Ali from Delhi
```

- `call()` → arguments passed individually.
- `apply()` → arguments passed as an array.
- `bind()` → returns a new function with `this` permanently bound.

## 32. Recursion

A function calling itself.

```javascript
function factorial(n) {
  if (n === 1) return 1;
  return n * factorial(n - 1);
}

console.log(factorial(5));
```

**Output:** `120`

## 33. Function Execution Flow (Call Stack)

```javascript
function first() {
  console.log("First");
}

function second() {
  first();
  console.log("Second");
}

function third() {
  second();
  console.log("Third");
}

third();
```

**Execution order:**

```
third()
  ↓
second()
  ↓
first()
```

**Output:**

```
First
Second
Third
```
